This Bulletin issue is devoted to the program equipment for emission measurements processing and evaluation. 

ECM traditionally lies great emphasis on progressive solution of program products. Introduced program equipment represents the first member of new generation of object orientated ECM products aimed at Windows NT environment.

 

Basic ECM CEM program characteristics
  • full compatibility with legislation and simple modifications implementation in case of its contingent amendment
  • own company development system allowing for large flexibility in customer requirements realization - presentation and way of evaluation
  • program equipment working in operation system Windows NT 4.0
  • system openess - from the point of view of successive increase of number of measuring signal sources on one hand and heterogeneity of their communication protocols on the other
  • high database reliability
  • easy definition of the way of measured and recorded data processing (criteria and mathematical formulas definition)
  • easy creation of print-out and graphic outputs
  • possibility of connection to plant computer networks
  • processed and archived data access for authorized external users

Program equipment ECM CEM takes in full extent above mentioned criteria into account. Its basis, D2000 software, is compatible with the software used in the systems "Transit gas pipeline monitoring and control", "Nováky power plant information system" or installation in nuclear power plant Jaslovské Bohunice.

System architecture can be defined as follows :

  • signal source - analyzer / sensor / transducer
  • control and datalogging system at measuring site - PC / PLC / datalogger
  • computer server
  • operator consoles - local / network / distant operator

Server communicates with analyzers directly, through dataloggers, PC or PLC, and provides also for connection to local computer networks. Authorized user gains access to immediate data as well as to archived data (raw, calculated or processed data according to different criteria) either directly on a server console or via distant console installed in a local network, or through modem transfer provided by commuting link. On the operator console the same presentation program equipment as on the server is being installed. In such a way each user in the network has the same comfort as the server operator. The same pays for distant authorized users. The only difference is, that data are archived only on the server. It should be pointed out, that all data do not have to be processed only according to one key. Data coming from emission monitoring, water monitoring or imission monitoring station can be processed in different and unique way. Each group according to specific requirements and respective legislation.

The most important we consider to be a subsystem for emission monitoring data processing. The reason is relatively large number of data processed in real time. There can be various files of mathematical interchannel calculations as well as treatment of numerous status signals. Extraordinarily strict is also the requirement for minimum 95 % system reliability. This reliability comprises failures of every kind - instruments malfunction as well as data processing failure. From the point of view of reliability the most advantageous is attitude, where dataloggers are used between analyzers and the server. There are of course also other, less reliable possibilities. Level of processing and control between analyzer and server

In case of simple systems we prefer direct PC (it does not even need to have character of server) connection with an analyzer. It is advantageous, if the analyzer allows for direct communication via serial output. This eliminates need for double A-D and D-A conversion. PC is responsible for calculations and datalogging. However, in the case of PC drop-out measured data will be lost.

In case the analyzer is not equipped with a serial output, conversion cards have to be installed into the PC. Monitoring of larger enterprises is much more difficult, however. Individual PC or dataloggers have to be used for data preprocessing. The task of PC or DL in monitoring systems is to sense immediate values, calculate minute and 30 minute averages, to archive them, evaluate validity of sensed immediate values (taking into account analyzers functionality or validity level of measured parameters) and validity of mean half-hour averages. Access to data is secured by system of passwords.

DL is a data microprocessor system, suggested for data reception, control, logging and archived data print-out in the form of tables and graphs. It is a "black box" of the system, in which once sensed, calculated and archived values cannot be changed by no means.

From the hardware point of view DL is designed as a processor board with memory cooperating with modular architecture of interface cards, which can be configured as follows :

  • analogue input card with 8 voltage inputs
  • analogue input card with 8 current inputs
  • card with 8 digital inputs
  • card with 8 digital outputs
  • card with 4 analogue outputs

DL firmware provides for the following operations in monitoring systems:
  • continuous monitoring of analogue signals from analyzers and their conversion from analogue to digital form by 14-bit AD converter
  • evaluation of immediate values validity taking into account mask
  • digital inputs signaling system status
  • calculation of 1 minute averages from valid immediate values
  • evaluation of 1 minute averages validity
  • calculation of 30 minute averages from valid 1 minute averages
  • evaluation of 30 minute averages validity
  • 30 minute averages archiving
  • print out of immediate and archived values on terminals
  • system calibration control and creation of calibration print outs

DL monitors immediate values of analyzer analogue outputs being connected to DL analogue inputs. At the same time DL monitors its own digital inputs by means of which correct operation of the respective analyzer is being monitored, and thus determined validity or non-validity of measured data.

Digital inputs monitor also information on the status of monitored technology. In case of fulfilment of immediate value validity it considers such a value as a basis for basic 1 minute interval calculation. 1 minute averages are calculated as an arithmetic average of all immediate values within the respective minute. 1 minute average is considered to be valid for 30 minute average calculation, if all of its representative immediate values are valid.

DL further calculates mean half-hour values as arithmetic averages from 1 minute averages of individual monitored parameters. Mean half-hour average is considered to be valid, if more than 67 % of 1 minute averages is valid. In DL a valid half-hour average is marked with the sign "> ", while a non-valid half-hour average is marked with the sign "< ". DL allows for retrospective viewing of 1 minute averages and 30 minute averages of archived parameters. It means, that even after interruption of communication between PC and DL the program equipment retrospectively reads all missing data after renewing of the connection. Archiving capacity of 1 minute averages is limited to 2 hours. In case of power failure data and configuration memory is backed-up in DL for more than 2 months.

Data processing on server and thei server / console presentation

Software in PC is designed to perform the following tasks in a monitoring system:

  • read 1 minute and 30 minute averages
  • store and archive measured and calculated values in electronic form during required time
  • create daily, monthly and anual print-outs
  • evaluate daily values and 30 minute averages in order to judge emission limit
  • evaluate emission levels of monitored parameters
  • monitor mass flow of gaseous compounds and solid particulates
  • allow electronic data transfer in real time to centers determined by legislation

Server works under Windows NT operation system. Data are stored into 2 SQL databases. The first one is working database, the second one is safety database. Safety database is stored on the second computer hard disk. Program is protected by means of safe system of passwords. Administrator assigns access rights to other system users. Software allows to create data files on a diskette or CD-ROM disc via CD Writer.
